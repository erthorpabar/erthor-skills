# 根本区别
process - threa - coroutine
进程 - 线程 - 协程
内存等硬件 - cpu计算资源 - 交替执行(程序控制 等待时间做点别的)

# 概念1
并行 = 同时执行 = 多进程 多线程
并发 = 交替执行 = 多协程

# 概念2
异步 = 等待结果才能继续 = 串行 = 顺序执行
异步 = 等待时候做点别的 = 多进程 多线程 多协程

# 使用场景
占有资源多                                  => 多进程(分布式计算 = 多gpu部署大模型)
占有资源不多 -> 计算时间多                   => 多线程(cpu密集 并行同时执行 = 复杂计算)
占有资源不多 -> 计算时间不多 -> 需要等待      => 多协程(in/out密集 并发交替执行 = 网络请求 磁盘读写)


# 并行锁 
# 多进程 多线程
避免同时修改同一份数据
1 原子锁 = 要么全执行 要么都不执行
2 互斥锁 = 互斥访问 只有一个能访问 其他线程需要等待锁释放
3 读写锁 = 可以同时读和写 但改需要锁
