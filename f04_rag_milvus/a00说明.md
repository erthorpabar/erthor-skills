=====名词解释=====
- fsn
完全限定名称 代表最标准化的术语 可包括不同语言

- snomed ct id
唯一id

- IS_A
从属关系
心肌梗塞 父类包括 心脏疾病
糖尿病 子类包括 2型糖尿病

- 属性
肺炎的属性关系
1 有发病部位 = 肺组织
2 有病因 = 肺炎链球菌(细菌性肺炎的情况)
3 有临床表现 = 发热 咳嗽 呼吸困难


阿司匹林的属性关系
1 有活性成分 = 乙酰水杨酸
2 有药理作用 = 抗血小板凝聚
3 有适应征状 = 预防心肌梗塞 缓解疼痛

=====数据格式=====
fsn      SCTID         IS_A                domain_id
糖尿病    73211009      属于代谢性疾病       disorder(代表属于疾病类别)


domain_id = 领域
concept_class_id = 类型

=====商业价值=====
口语 标准化 处理 提供的api都是付费的



=====应用场景=====

场景1 医疗记录标准化 -> 不同医疗系统可互相整合
患者出现胸痛，诊断为急性心肌梗死
急性心肌梗死 -> 22298006


场景2 多语言环境 -> 不同语言可互相整合
急性心肌梗死 -> 22298006
myocardial infarction -> 22298006


场景3 药物相互作用检测
已经服用华法林(抗凝药) 的患者开 阿司匹林 
识别到 两个药物都有 抗凝 作用
自动发出药物相互作用警告 可能增加出血风险


场景4 医疗数据分析
分析所有心脏疾病相关数据
通过is_a 自动识别 心肌梗死 心绞痛 心律失常 等子类


场景5 临床决策支持
通过症状
发热 咳嗽 血常规白细胞升高
通过关系网络 -> 可能为 肺炎 支气管炎 从而缩小疾病范围 并推荐检测和治疗方案
