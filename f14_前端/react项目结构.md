# 各文件功能
├── index.js 把app组件渲染到页面中
├── App.js 整合所有组件到app组件中
│ 
├── components/
│   ├── xxx.js # 各种组件
│ 
├── package.json # 项目描述文件
│   ├── "scripts" # 定义各种启动命令
├── lock.json 或 lock.yaml # 依赖包版本锁定

# ——————————调用逻辑————————————
index.html （webpack 自动寻找 index.js）
    -> index.js （渲染 虚拟dom -> 真实dom）
        -> App.js （整合其他组件）
            -> components/xxx.js （组件）
                -> 组件 = html + css + js 的集合
                -> 组件只提供一种功能
                -> 组件可以被复用


# ——————————项目书写思路——————————
0 所有组件都定义为函数
1 组件与return渲染位置相对应
2 交互循环(交互->触发事件->改变状态->重新计算->重新渲染)
3 标注运行顺序(ui线程与子线程)
4 性能控制，卸载组件时卸载模型，并确保模型只加载一次

# ——————————react特性————————————
1 node = 节点 = <a>标签</a>
2 dom = 一大堆node
3 虚拟dom = 每次只更新变化的部分 ( 虚拟dom不存储 属性的value值)
4 所有虚拟dom，都通过js来书写
5 虚拟dom中的js = jsx 
    引用变量，函数，需要加{}
    很多固定标签名称被重写了一遍

    # ——————————组件间通信————————————
1 上级→ 下级
[上级 通过 props 向下级传递数据]
上级 标签<a = 10> 输入
下级 props 接收 

2 下级A → 上级 → 下级B
[上级组件维护 变量 和 改变变量的函数，下级组件 绑定触发事件，并接收变量改变]
上级 1变量，2改变变量的函数
下级A props接收函数，函数绑定触发事件
下级B props接收变量，展示变量
